# claude-spawn 口播词脚本

## 1. Hook / 开场（~15 秒）

大家好，今天给大家看一个我做的小工具，叫 claude-spawn。它让你同时跑多个 Claude Code session，共享上下文，文件互不干扰。

我现在在 Claude Code 里面，手上有一个正在进行的任务。我想同时探索三个不同的方向。我只需要敲一行命令——

（切屏幕：敲 /fork，三个 pane 同时打开）

就这样，三个独立的 session 就出来了。每一个都继承了我刚才完整的对话历史，而且各自有自己的 git worktree，互不干扰。

## 2. 痛点共鸣（~20 秒）

用过 Claude Code 的朋友应该都有这个体会：你在一个 session 里面聊了很久，积累了大量上下文，这时候你想起来，另一件事也需要处理。

怎么办呢？开一个新终端，再启动一个 Claude，然后把之前的背景信息复制过去——写个 handoff doc，或者干脆手动复制粘贴。你变成了两个 AI 之间的传话筒。

更麻烦的是，如果两个 session 同时在改文件，它们会互相踩。你还得处理冲突。

## 3. 功能介绍（~40 秒）

claude-spawn 就是解决这个问题的。它有两个命令。

第一个，/new。你告诉它新 session 要做什么，它会把当前对话里相关的上下文自动蒸馏成一个 prompt，然后在新的 iTerm2 窗口里启动一个干净的 session。你也可以让它先把要传过去的内容列出来，你确认之后再发。不需要你手动写 handoff，不需要复制粘贴，一步到位。

第二个，/fork。这个更直接——你用自然语言描述你想并行做哪些事，它就会把当前的会话 fork 成对应数量的 pane。每个 pane 拿到的是你完整的对话历史，从你离开的地方接着干。

关键是，不管是 /new 还是 /fork，每个新 session 都跑在自己的 git worktree 里。你的主分支不受影响，并行的 agent 之间也不会互相踩文件。

如果你只是想让它先做调研不急着动手，还可以加 --plan 让新 session 进入 plan mode。

所以简单来说，claude-spawn 让你的 Claude Code 从一次只能做一件事，变成可以同时推进多条线。上下文自动传递，文件隔离不冲突。
